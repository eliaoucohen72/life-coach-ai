# PRD — AI Life Coach (Sport, Nutrition, Wellbeing)

## 1. Vision

Build a web application that provides users with a personalized AI coach covering sport, nutrition, and quality of life. The coach engages via a conversational chat interface, remembers user context within a session, and adapts its tone and advice based on a user profile.

---

## 2. Personas

### Persona A — Yoni, 32, software developer
- Sedentary job, wants to lose weight and build a habit of exercise
- No nutritionist or trainer, needs accessible guidance
- Uses the app on desktop during lunch break or evenings

### Persona B — Sarah, 27, freelancer
- Already active, wants to optimize nutrition and sleep quality
- Motivated, needs structure and accountability
- Uses the app on mobile, short sessions

### Persona C — David, 45, manager
- Stress management and work-life balance as primary goals
- Some knowledge of nutrition, wants quick actionable advice
- Occasional use, needs the app to remember his context

---

## 3. User Stories

### Onboarding
- As a new user, I want to fill in my profile (age, gender, weight, goals, dietary restrictions, activity level) so the coach knows who I am
- As a user, I want to skip onboarding and come back to it later

### Chat
- As a user, I want to send messages and receive coach responses in real time (streaming preferred)
- As a user, I want the coach to remember what I told it earlier in the conversation
- As a user, I want to start a new conversation and keep past ones accessible

### Profile
- As a user, I want to view and edit my profile at any time
- As a user, I want my profile to influence the coach's advice automatically

### History
- As a user, I want to scroll through past conversations
- As a user, I want to delete a conversation

### Multilingual
- As a user, I want to use the app in French, Hebrew, or English
- As a user, I want the coach to respond in the same language I write in

---

## 4. Features

### MVP (Phase 1)
- User profile creation (local storage)
- Chat interface with streaming responses
- Conversation history (local storage)
- Coach persona: motivating, warm, evidence-based
- Language auto-detection and response matching
- Mobile-responsive UI

### Phase 2
- User authentication (JWT)
- Persistent storage (PostgreSQL or MongoDB)
- Multiple coach modes: strict / supportive / neutral
- Weekly summary / progress notes generated by the coach
- Push notifications (PWA)

### Out of scope
- Wearable integrations
- Calorie tracking with food database
- Video content

---

## 5. Coach Persona

**Name:** Flex (flexible, adaptable)

**Personality:**
- Warm and encouraging by default
- Direct and honest when the user asks for tough feedback
- Science-grounded (no pseudoscience)
- Adapts tone to user mood detected in messages
- Never judges. Never shames.

**Domains:**
- Sport: training plans, movement, recovery
- Nutrition: balanced eating, hydration, meal timing
- Quality of life: sleep, stress, habits, mindset

**System prompt base:**
```
You are Flex, a personal AI coach specializing in sport, nutrition, and quality of life.
You are warm, motivating, and evidence-based. You never shame the user.
You adapt your tone to the user's mood. You respond in the same language the user writes in.
You always take the user's profile into account when giving advice.
User profile: {profile}
```

---

## 6. UX/UI Direction

- Clean, energetic design. Not clinical, not gym-bro.
- Dark mode default with an optional light mode toggle
- Color palette: deep navy + electric green accent + warm white text
- Typography: strong display font for coach name/headers, readable sans-serif for chat
- Chat bubbles: user on the right (muted), coach on the left (accented)
- Subtle typing animation during streaming
- Onboarding as a friendly conversational flow, not a form

---

## 7. Success Metrics

- User completes onboarding: > 70%
- User sends 3+ messages in first session: > 50%
- User returns for a second session: > 30%
- Coach response time: < 2 seconds (Groq LPU target)

---

## 8. Risks

| Risk | Mitigation |
|---|---|
| Groq free tier rate limits | Add retry logic + user-facing message |
| Medical advice liability | Add disclaimer on first use |
| Profile data privacy | Local storage only in MVP, no PII sent to third parties |
| Language detection failures | Fallback to English, allow manual override |
